SPRING BOOT FRAMEWORK y SPRING BOOT

SPRING BOOT FRAMEWORK

Es un framework Open Source que ayuda a crear aplicaciones en Java, Kotlin, Maven y Groovy.


La inyección de dependencias, es uno de los pilares que tiene Spring Framework, aunque está dividido en diferentes módulos dando buenas funcionalidades contributivas:


  • Container: suministra la inyección de dependencias e inversión de control.
  • Web: facilita la creación de controladores Web.
  • Acceso a datos: gestiona sobre JDBC, ORMs como Hibernate, sistemas OXM (Object XML Mappers), JSM y transacciones.
  • Programación orientada a Aspectos (AOP): da buen soporte para los aspectos.
  • Instrumentación: ofrece un buen soporte para la instrumentación de clases.
  • Pruebas de código: contiene un framework de testing, para JUnit y TestNG y lo que haga necesario para testear los mecanismos de Spring.

Los módulos son opcionales, no es necesario llenar el classpath con clases que no se van a utilizar. Solo usa lo que debe usar.


La configuración inicial y preparación de las aplicaciones diseñadas por medio de Spring Framework para producción son algo complejas. Pero Spring Boot simplifica el proceso por medio de dos principales mecanismos, para darle mayor provecho.


SPRING BOOT

Spring Boot permite compilar aplicaciones Web como un archivo .jar que se ejecuta normalmente como cualquier aplicativo.


Spring boot integra el servidor de aplicaciones en el propio .jar y despliega cuando se ejecuta la aplicación. Así se puede distribuir las aplicaciones de una forma más sencilla, configurando el servidor junto con la aplicación. Es bastante útil en arquitecturas de microservicios, ya que permite distribuir las aplicaciones como imágenes Docker que se puede escalar de manera horizontal.


Spring Boot ofrece una serie de dependencias, que se puede añadir a los proyectos, pero teniendo cuenta solo lo que se necesita: Se puede crear un controlador REST, acceder a una base de datos utilizando JDBC, conectar con una cola de mensajes Apache ActiveMQ, etc.


Cuando se añade un starter, éste ofrece todas las dependencias necesarias, tanto de Spring como de los terceros. Además, los starters ya están configurados con valores por defecto, que reduce la configuración cuando se va a trabajar. El spring-boot-starter-actuator es un buen ejemplo que ayuda a añadir dependencias para los proyectos.

Así como Spring Framework, se puede realizar cualquier configuración para modificar lo necesario posible.
